> **Generated from source code analysis of `d:\nutridiet` (Post-Cleanup/Merge State)**

## 1. Project Purpose

**NutriDiet** is an AI-powered, personalized health and nutrition management web application built with Django. It solves the problem of generic, one-size-fits-all diet advice by combining:

- **Real nutritional science** (Harris-Benedict Equation for BMR/TDEE, BMI classification)
- **Generative AI** (Google Gemini 2.0 Flash) for personalized meal plan creation and natural-language coaching
- **Machine Learning** (Scikit-Learn linear regression) for predictive weight trend analytics
- **Professional reporting** (ReportLab PDF + OpenPyXL Excel) including embedded charts

The application is culturally specialized for **Indian cuisine** and respects dietary preferences (Vegetarian, Non-Vegetarian, Vegan), food allergies, medical conditions, and restrictions (Gluten-Free, Keto, Halal, Jain, etc.).

Target audience: Health-conscious individuals in India who want a scientifically grounded, AI-assisted system to track nutrition, manage body weight, and receive personalized guidance.

## 2. Stakeholders / User Roles

Three distinct roles are implemented, each with separate access paths:

| Role | Definition | Access Path |
|------|-----------|-------------|
| **Anonymous Visitor** | Unauthenticated user | Landing page only (`/`) |
| **Regular User** | Authenticated user with a `UserProfile`. Two sub-tiers exist: **Free** and **Pro** | `/dashboard/`, `/logs/`, `/coach/`, `/diet/`, etc. |
| **Platform Admin** | Django `User` with `is_staff=True` | Custom admin site at `/admin/` (NutriDietAdminSite) |

### User Sub-tiers (defined in `UserProfile.subscription_status`)
- **Free** — Standard access; AI diet plan is generic (no specific caloric targets)
- **Pro** — Paid subscriber; receives highly tailored AI plans with exact weight/calorie targets, access to the Advanced Diet Planner, Full Report page, and PDF/Excel export

## 3. Features by User Role

### 3.1 Anonymous Visitor
- View the landing/marketing page (`/`)
- Register a new account with OTP email verification (`/register/`, `/api/send-otp/`)
- Login (`/login/`)
- Password reset flow (`/password-reset/` → confirm → complete)

### 3.2 Regular User (Free Tier)
- **Dashboard** — Overview: today's calorie intake vs. target, BMR/TDEE stats, macros breakdown, water intake tracker (glass-by-glass), weight records, recent activity feed, AI-generated status summary
- **Food Logging** — Add manual food consumption logs from the food item database (breakfast/lunch/dinner/snack)
- **Weight Tracking** — Record daily weight; view history
- **AI Coach** — Natural-language chat interface powered by Gemini 2.0 Flash (asks health/recipe/grocery questions)
- **AI Diet Plan (Generic)** — Daily Indian diet plan generated by Gemini, respecting dietary preference and allergies. Plan saved to database.
- **Logs** — Paginated feed showing the last 20 food entries, grouped by date and meal type
- **All Logs** — Unlimited log view
- **Favorites** — Save meals/foods as favorites by day-of-week and meal type (heart toggle)
- **Settings / Profile** — Update personal metrics (name, age, gender, height, weight, target weight, activity level, dietary preference, allergies, medical conditions, diet restrictions, profile image URL); change password; change email
- **Billing** — View subscription plans, transaction history, upgrade to Pro (UPI, Net Banking, Card)

### 3.3 Regular User (Pro Tier)
All Free features, plus:
- **AI Diet Plan (Tailored)** — Gemini generates a plan analyzing exact weight/target/activity data; Pro-tier context passed in prompt
- **Advanced Diet Planner** (`/diet-planner/`) — Richer UI allowing submission of detailed profile data (goals, meals per day, disease conditions, cuisine preference) for a structured JSON medical-grade plan (includes BMI assessment, calorie targets with macros, avoid foods, superfoods, hydration tips, lifestyle tips, medical alerts)
- **Full Report** — Detailed on-screen health report with: calorie stats, weight trends, ML prediction, AI executive summary, recent logs and water records
- **PDF Export** — Download a PDF report with consumption table, weight/hydration history table, embedded weight trend chart, embedded water intake bar chart, AI summary
- **Excel Export** — Download an `.xlsx` report with Executive Summary sheet, Consumption Logs sheet, Weight & Water History sheet
- **Invoice Download** — PDF invoice per transaction

### 3.4 Platform Admin (`is_staff=True`)
- **Custom Admin Dashboard** — KPI cards: total users, new users this week, Pro users & conversion rate, total revenue, total expenses, net profit, food items count, logs today, water/weight/diet-plan logs today, deleted item count
- **Dashboard Charts** — 7-day registration trend (line), meal-type distribution (doughnut), expense-by-category (bar) — rendered server-side in Chart.js
- **PDF Admin Report** — Downloadable PDF (`/admin/dashboard-report/`) with date-range filtering, Revenue vs Expenses bar chart, Deleted Items pie chart, recent user list
- **Database Management** — CRUD interfaces registered via `NutriDietAdminSite`:
  - `User` & `Group` (Django auth)
  - `UserProfile` (with fieldsets for Account, Body Metrics, Nutrition Preferences, Subscription)
  - `FoodItem` (nutritional data for the food catalog)
  - `ConsumptionLog`, `WeightRecord`, `WaterLog`, `DailyMealLog`, `DailyDietPlan`
  - `SubscriptionPlan` (fully configurable — amount, features list, badge, duration, display order)
  - `Transaction` (read/filter payment records)
  - `FoodPreference` (user favorites)
  - `AdminExpense` (Infrastructure, Marketing, Operations, Software, Other)
  - `DeletedRecord` (audit trail of all soft-deletions via Django signals)

## 4. Workflow — End-to-End Flow

```
[1] VISITOR → Landing Page (/)
     │
     ├─ [2] REGISTRATION
     │       • Fill username + email + password
     │       • JS POSTs to /api/send-otp/ → server validates uniqueness,
     │         generates 6-digit OTP, emails it via Gmail SMTP
     │       • User enters OTP in form → POST /register/
     │       • Server validates OTP (10-min expiry) → creates Django User
     │       • Auto-creates UserProfile (defaults: Male, 25y, 170cm, 70kg)
     │       • Auto-login → redirected to /dashboard/
     │
     ├─ [3] LOGIN (/login/)
     │       • Username + Password → Django authenticate()
     │       • "Remember Me" checked → session expires in 2 weeks
     │       • "Remember Me" unchecked → session expires on browser close
     │       • Staff users → redirect to /admin/
     │
     └─ [4] PASSWORD RESET (Django auth views for reset/confirm/complete)

[5] DASHBOARD (/dashboard/)
     • calculate_dashboard_stats() computes BMR (Mifflin-St Jeor), TDEE,
       daily_calorie_target (TDEE − 500 if overweight), macros
     • ML: predict_weight_trend() trains LinearRegression on WeightRecord history
       → 7-day forecast weight, trend (Upward/Downward/Stable), R², goal date
     • AI: generate_report_summary() calls Gemini → short executive summary
     • WaterLog get_or_create for today with calculated target glasses
     • Render: stats, logs, weight chart data, water tracker, activity feed

[6] FOOD LOGGING
     • User selects food item from FoodItem catalog, specifies meal type & quantity
     • POST /api/add-log/ → creates ConsumptionLog (FK to UserProfile + FoodItem)
     • Same-page stats refresh via AJAX

[7] WEIGHT TRACKING
     • POST /api/add-weight/ → creates/updates WeightRecord for the date
       (unique_together: user_profile + date)
     • Also updates profile.weight to latest reading

[8] WATER TRACKING
     • POST /api/add-water/ (action: add | remove)
     • Updates WaterLog.amount_glasses for today
     • Returns JSON with completion status

[9] AI DIET PLAN (/diet/)
     • Checks for existing DailyDietPlan for today; creates if absent
     • build Gemini prompt with: BMI, dietary preference, allergies, medical conditions,
       restrictions, Free/Pro tier context
     • Gemini returns JSON → parsed, blocked ingredients checked (2 retries)
     • Plan saved to DailyDietPlan model
     • User can log individual meals from plan to DailyMealLog via /api/log-meal/

[10] ADVANCED DIET PLANNER (/diet-planner/) — Pro Only
      • User fills detailed form: goal, diet type, activity, diseases, cuisine, meals/day
      • POST /api/diet-plan/ → generate_diet_plan() → Gemini with build_diet_plan_prompt()
        → disease-specific rules (Diabetes=Low GI, Hypertension=Low Na, PCOS, IBS, etc.)
      • Returns structured JSON: bmi_assessment, calorie_target, meal_plan[], avoid_foods[],
        superfoods[], hydration, lifestyle_tips, medical_alert
      • If logged-in Pro user: save_advanced_diet_to_db() persists to DailyDietPlan

[11] AI COACH (/coach/)
      • User types natural-language question
      • POST /api/coach/ → builds context string with profile + recent 10 logs
        (Pro: also includes weight & target) → calls Gemini 2.0 Flash
      • Returns AI response displayed in chat UI

[12] REPORTS & EXPORTS
      • /report/ (Pro only) — on-screen full report with ML predictions + AI summary
      • /export/report/?format=pdf&start_date=...&end_date=... — calls:
          export_to_pdf(): ReportLab PDF with charts (matplotlib)
          export_to_excel(): OpenPyXL .xlsx
      • /api/download-invoice/<transaction_id>/ — ReportLab PDF invoice

[13] SETTINGS (/settings/)
      • action=update_profile → update UserProfile + User.email
      • action=change_password → Django PasswordChangeForm + session re-hash

[14] BILLING (/billing/)
      • Displays SubscriptionPlan records (admin-configurable)
      • POST /api/process-payment/ → creates Transaction, sets profile.subscription_status='Pro',
        calculates new expiry date (stacks from current expiry if not expired)

[15] FAVORITES (/favorites/)
      • Heart-icon toggle → POST /api/toggle-favorite/ (food_name, meal_type, day_of_week)
      • Creates/toggles FoodPreference.is_favorite
      • Favorites displayed on /favorites/ page and shown as hints on diet plan page

[16] LOGOUT (/logout/)
      • Django logout + session flush → redirect to landing
```

## 7. AI & Special Modules

### 7.1 Google Gemini API (`google-generativeai`)

**Model used:** `gemini-2.0-flash` throughout  
**API key:** read from `.env` → `GEMINI_API_KEY` → `settings.GEMINI_API_KEY`

| Function | Location | Description |
|----------|----------|-------------|
| `generate_indian_diet()` | `api/ai_utils.py` | Primary diet generator. Builds a prompt with BMI, dietary pref, allergies, medical conditions, restrictions, Pro/Free tier context. Gets JSON back and saves to `DailyDietPlan`. Retries once if blocked ingredients are detected. |
| `generate_report_summary()` | `api/ai_utils.py` | Creates a 3–4 sentence AI health summary for use in the dashboard, report page, and report exports. |
| `generate_diet_plan()` | `api/ai_utils.py` | Advanced planner used by `/api/diet-plan/`. Prompts Gemini for a structured JSON with disease-specific rules (Diabetes, Hypertension, Thyroid, PCOD, IBS, Heart Disease, Kidney Disease). Returns `bmi_assessment`, `calorie_target`, `meal_plan[]`, `avoid_foods[]`, `superfoods[]`, `hydration`, `lifestyle_tips`, `medical_alert`. |
| `get_water_recommendation()` | `api/ai_utils.py` | Queries Gemini for a personalized daily water intake goal (ml + glasses + rationale). Falls back to `weight × 35ml` formula if API unavailable. |
| `ai_coach_api()` | `user_app/views.py` | Direct Gemini call for the chat coach. Sends user profile context + recent meals in the prompt. |

**Safety validation in `ai_utils.py`:**
- `DIET_RULES` dict — blocked terms for `Veg` and `Vegan` preferences
- `RESTRICTION_RULES` dict — blocked terms for `gluten-free`, `keto`, `halal`, `jain`
- `_blocked_ingredient_terms()` — merges allergy tags + diet rules + restriction rules into a set
- `_plan_has_blocked_items()` — scans any generated meal text for blocked terms; triggers re-generation

### 7.2 Machine Learning — Scikit-Learn (`api/ml_utils.py`)

**Function:** `predict_weight_trend(user_profile, days_ahead=7)`

- Requires ≥ 3 `WeightRecord` entries
- Trains a `sklearn.linear_model.LinearRegression` model on `(days_since_start, weight)` pairs
- Outputs:
  - `predicted_weight` — forecast 7 days ahead
  - `attainment_date` — estimated date to reach `target_weight` using `x = (target - intercept) / slope`
  - `confidence_score` — R² value
  - `trend` — `Upward` (slope > 0.05) / `Downward` (slope < -0.05) / `Stable`
  - `slope`, `intercept`, `message`

### 7.3 ReportLab (`reportlab`) — PDF Generation

Used in two places:

1. **`api/report_utils.py` → `export_to_pdf()`** — User health reports with:
   - Title + AI summary paragraph
   - Weight & Hydration History table (styled with `TableStyle`)
   - Weight trend chart (Matplotlib → PNG → embedded via `Image`)
   - Water intake bar chart (Matplotlib → PNG → embedded)
   - Food Consumption Logs table
2. **`user_app/views.py` → `download_invoice_api()`** — Minimal PDF invoice per transaction
3. **`admin_app/admin_views.py` → `dashboard_report_view()`** — Admin performance PDF with financial charts

### 7.4 OpenPyXL (`openpyxl`) — Excel Generation

**`api/report_utils.py` → `export_to_excel()`** — Produces `.xlsx` with:
- Sheet 1: Executive Summary (AI summary text)
- Sheet 2: Consumption Logs (Date, Type, Details, Calories)
- Sheet 3: Weight & Water History (Date, Weight, Water Glasses, Goal Met)

### 7.5 Matplotlib (`matplotlib`)

Used in `report_utils.py` and `admin_views.py`:
- Non-interactive backend (`Agg`) for server-side chart generation
- Weight trend line chart (with fill)
- Water intake bar chart
- Admin: Revenue vs Expenses bar chart
- Admin: Deleted Items pie chart

### 7.6 Django REST Framework (`djangorestframework`)

- `ViewSet` classes in `api/views.py` for CRUD on profiles, food items, consumption logs, weight records
- `DefaultRouter` in `api/urls.py`
- Serializers: `UserProfileSerializer`, `FoodItemSerializer`, `ConsumptionLogSerializer`, `WeightRecordSerializer`, `UserProfileListSerializer`

- `RegistrationOTP` model stores the code with a 10-minute TTL
